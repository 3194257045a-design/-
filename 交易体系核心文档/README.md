# 交易体系核心文档

> 构建系统化、科学化的交易知识体系

## 文档架构说明

这是所有交易相关文档的顶层收纳目录，采用**纯文件夹+导航文件**的架构设计，实现知识体系的系统化管理。

### 三大核心维度

1. **🎯 [[仓位管理维度导航]]** - 风险控制的核心
   - 资金配置、风险敞口管理、仓位策略优化
   
2. **⚙️ [[交易系统维度导航]]** - 交易执行的框架
   - 系统设计、买卖规则、风险监控机制
   
3. **📊 [[市场分析维度导航]]** - 决策判断的依据
   - 分析方法、行情判断、市场情绪分析

### 文档层级逻辑

```
顶层：README.md（本文件）
├── 维度层级：3个维度导航主文件
├── 模块层级：12个模块导航主文件（每个维度4个模块）
└── 主题层级：48+个具体主题笔记文档
```

- **维度导航文件**：负责模块间的逻辑组织和引导
- **模块导航文件**：负责主题间的层级管理和内容索引
- **主题笔记文件**：承载具体知识内容和实践指南

## 使用说明

### 文件命名规范

- **导航文件**：以"导航"结尾，如`仓位管理维度导航.md`
- **主题文件**：具体内容文档，如`仓位定义.md`
- **模块文件**：按`模块序号-模块名称`格式组织

### 内部链接格式

- **维度间链接**：`[[维度名称导航]]`
- **模块间链接**：`[[模块序号-模块名称]]`
- **主题间链接**：`[[主题名称]]`

### 维度间关联

三大维度相互支撑，形成完整交易闭环：

```
市场分析 → 决策判断 → 交易系统 → 执行操作 → 仓位管理 → 风险控制
```
